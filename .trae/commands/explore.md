# /explore — 轻量代码探索

当用户使用 `/explore <目标>` 时，快速探索指定模块/文件/功能，直接输出分析结果，不创建工作流目录。

## 与 /research 的区别

| | `/explore` | `/research` |
|---|---|---|
| 工作流 | 不创建 | 创建完整工作流目录 |
| 产出 | 口头/对话内输出 | 写入 `repo-exploration.md` 文件 |
| 深度 | 快速概览（3-5分钟） | 深度调研（10个章节结构化报告） |
| 适用 | 快速了解某个函数/模块/文件 | 正式接手项目前的完整摸底 |

## 流程

### 第一步：确定探索范围

1. 如果用户指定了文件/模块/函数 → 聚焦该范围
2. 如果用户描述了功能 → 定位相关代码后探索
3. 如果范围过大 → 询问用户缩小范围

### 第二步：快速探索

调用 @code-explorer 进行轻量探索。

探索内容：
- 目标模块的文件结构和入口点
- 关键函数签名和调用关系
- 依赖项和影响面
- 现有模式/规范

### 第三步：向用户呈现

直接在对话中用 5-10 句话概括发现：
- 模块结构概览
- 关键入口和调用路径
- 需要注意的约束或风险

---

## 约束
- 不创建 `.specdev/` 目录
- 不写入 `active-workflow`
- 不产出文件（除非用户明确要求写报告）
- code-explorer 只读，不修改代码
