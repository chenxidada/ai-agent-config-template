# idea-to-mvp

Use this workflow when a user starts from a product idea.

1. Ask `repo-explorer` to inspect the existing repository or surrounding system constraints before planning
2. Ask `requirement-analyst` to clarify goals, users, MVP, non-goals, and open questions
3. Ask `task-planner` to break the result into modules and vertical slices
4. Ask `solution-architect` to propose the technical design for the MVP only
5. Ask `knowledge-manager` to save the requirement and architecture milestones

Expected outputs:

- repository exploration
- requirement definition
- MVP task plan
- solution design
- knowledge-base summary
